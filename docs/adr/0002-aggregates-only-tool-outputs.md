# Aggregates-only tool outputs (no row-level data to the LLM)

Tool outputs are restricted to column-level aggregates — schema, summary statistics, value counts, correlation values, outlier counts — and never include a row that ties multiple fields together. This is deliberately stricter than allowing even a small raw-row sample, since a handful of rows can still re-identify individuals through quasi-identifier linkage, and a general-purpose agent over arbitrary user-supplied CSVs has no way to know upfront whether a given file is subject to HIPAA/GDPR/GLBA-style disclosure rules. It also closes a prompt-injection surface — raw, attacker-controllable cell text never flows into the LLM's context — and keeps the agent's local trace log low-sensitivity by construction, since tool outputs rather than raw data are what get logged.
